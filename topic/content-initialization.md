---
title: Content Initialization
---
This batch describes the default content that is copied into the system on first boot when the data directory is empty. It explains that the initial import seeds several core sections, including identity, profile, site metadata, landing page content, and example product pages. The purpose is to provide a starting structure that can be replaced later with organization-specific material. It also clarifies that after this first import, the database and git repository become the source of truth. This topic is important because it defines how a new instance is bootstrapped and where authoritative content lives afterward.
